Discuss the scope of var, let, and const

var, let, and const are JavaScript variable declarations that determine variable scope and mutability. var has global or function scope, whereas let and const are limited to block scope. While let permits variable reassignment, const enforces immutability. In contemporary JavaScript coding, it is advisable to default to const for constants and only opt for let when variable changes are necessary. To maintain code quality and prevent scope-related problems, it's best to steer clear of var.

Tell us the use cases of null and undefined

null and undefined are JavaScript values employed to denote the absence of meaningful data. Developers typically assign null explicitly to signify a deliberate lack of value. Conversely, undefined usually indicates a missing or uninitialized value. null is utilized to convey that a variable or object property should have no value, while undefined is more frequently encountered when a value is expected but not present.

What do you mean by REST API?

A REST API, which stands for Representational State Transfer Application Programming Interface, is a structured framework for building networked applications. It relies on specific rules and principles such as statelessness and a uniform interface to facilitate communication between clients and servers via HTTP. By utilizing standard HTTP methods like GET, POST, PUT, and DELETE, REST APIs enable clients to interact with server resources effectively. They are widely adopted for web services, streamlining data exchange between various software systems in an efficient and scalable manner.
